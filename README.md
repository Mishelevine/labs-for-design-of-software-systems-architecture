# Репозиторий по дисциплине "Проектирование архитектуры программных систем"

**Тема: Разработка программного обеспечения для автоматизации тестирования устройства вызова экстренных оперативных служб**

### Краткое описание:

Компания "FortTelecom" выпускает устройства для экстренного вызова оперативных служб, которые встраиваются в автомобили. При выпуске новой верии устройства или ПО для устройств их необходимо тестировать на соответствие существующим ГОСТам. Сегодня такое тестирование может занимать несколько недель.

Компания решила создать несколько специализированных стендов, содержащих все элементы устройства и необходимой инфраструктуры для автоматизации тестирования. Необходимо разработать ПО для управления стеном, а также проведения и мониторинга испытаний.

### Пользователи:
- Разработчики - возможность развития CI/CD, автоматическое тестирование при разработке новых версий ПО
- Тестировщики - возможность автоматического проведения испытаний на соответствие существующим ГОСТ, а также внутренним методикам испытаний компании. Автоматическое составление отчетов по итогам испытаний.
- Менеджмент и маркетинг - возможность просмотра статистики испытаний для маркетинговых и корпоративных целей

### Требования:
- Возможность написания автоматизированнных тестов на языке Python
- Логирование информации о проведении тестов и состоянии устройств
- Хранение и поиск логов всех элементов стенда и проводимых тестов в системе Loki
- Имитация и отслеживание неисправностей устройства при помощи блока реле (обрывов / имитаций короткого замыкания)
- Создание дашборда информирующего о текущем состоянии элементов стенда
- Автоматическое создание отчетов о проведенных испытаниях с возможностью настройки шагов, критериев выполнения и т.д.
- Интеграция системы в CI/CD GitLab компании

### Дополнительный контекст:
Сегодня все тесты и отчеты не автоматизированны, их выполняют тестировщики и разработчики при создании ПО  
Сегодня многие производители электорники уже используют автоматическое тестирование, что служит не только для ускорения разработки, но и в рекламных целях, так как клиенты могут увидеть, как выполняются испытания, что повысит доверие к компании.   
Стенд состоит из элементов самого устройства вызова экстренных служб, устройства для имитации точки приема экстренных вызовов, микрофона, блока для имитации неисправностей и выполнения измерений, а также устройства для выполнения тестовых скриптов и управления остальными элементами стенда.
